Wirepas Single-MCU SDK
API

API is split into following areas:

  • Libraries offered by stack. These are implemented in stack and header files are offered to access those.
  • SDK internal API. These are given as source code in SDK.
  • Board definitions, i.e. pinout mappings for various boards
  • Bootloader configuration, i.e. early initialization functionality
  • Low level hardware services (HAL), i.e. low level routines, such as access to peripherals
  • Utility and helper services, i.e. various utility- type functions, not related to stack operations as such, such as random number generation.
